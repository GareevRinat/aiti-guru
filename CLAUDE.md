# Aiti Guru - Тестовое задание (React Developer)

## Описание проекта
Административная панель управления товарами с авторизацией.
API: DummyJSON (Products + Auth).
Макет: Figma (см. `/ref/`).

## Технологический стек
- **React 18+** с TypeScript (strict mode)
- **Vite** — сборщик
- **React Router v6** — маршрутизация
- **TanStack Query (React Query)** — серверное состояние, кэширование, рефетч
- **React Hook Form + Zod** — формы и валидация
- **Zustand** — клиентское состояние (auth, UI)
- **Axios** — HTTP-клиент
- **SASS Modules** — стилизация вручную по макету Figma (без UI-библиотек, демонстрация навыков вёрстки)
- **React Hot Toast** — toast-уведомления

## React Best Practices (ОБЯЗАТЕЛЬНО)

### Архитектура
- Feature-Sliced Design (упрощённый): `features/`, `shared/`, `pages/`, `app/`
- Каждый компонент — в своей папке: `ComponentName/index.tsx`, `ComponentName.module.scss`
- Бизнес-логика отделена от UI через кастомные хуки
- Не допускать "God Components" — декомпозировать на мелкие переиспользуемые компоненты

### TypeScript
- `strict: true` в tsconfig
- Никаких `any` — использовать корректные типы, `unknown` при необходимости
- Интерфейсы для API-ответов, пропсов компонентов, форм
- Использовать `as const`, discriminated unions где уместно

### Состояние
- Серверное состояние — ТОЛЬКО через TanStack Query (не хранить API-данные в useState/Zustand)
- Клиентское состояние (auth token, UI флаги) — Zustand
- Не поднимать состояние выше, чем необходимо
- Мемоизация: `useMemo`, `useCallback` только при реальной необходимости (не преждевременная оптимизация)

### Компоненты
- Функциональные компоненты + хуки (никаких классов)
- Props destructuring в параметрах функции
- Ранний возврат (early return) для условного рендеринга
- `key` — стабильные уникальные значения (не индексы массива для динамических списков)
- Обработка состояний загрузки, ошибок, пустых данных для каждого запроса

### Формы
- Контролируемые формы через React Hook Form
- Валидация схемами Zod (не ручная проверка)
- Показ ошибок валидации под полями
- Disable кнопки отправки при `isSubmitting`

### API
- Базовый URL и конфигурация — в одном месте (`shared/api/`)
- Интерцепторы Axios для токена авторизации
- Обработка ошибок API централизованно
- Типизированные функции-запросы

### Безопасность
- Токены НЕ хранить в коде
- XSS: не использовать `dangerouslySetInnerHTML`
- Санитизация пользовательского ввода

### Производительность
- Ленивая загрузка страниц (`React.lazy` + `Suspense`)
- Дебаунс для поиска
- Пагинация на стороне API (не загружать все данные)

### Стиль кода
- Именование: PascalCase для компонентов, camelCase для переменных/функций, UPPER_SNAKE_CASE для констант
- Один компонент — один файл
- Хуки начинаются с `use`
- Осмысленные имена (не `data`, `item`, `temp`)

## Структура проекта
```
src/
  app/            # Провайдеры, роутер, глобальные стили
  pages/          # Страницы (Login, Products)
  features/       # Фичи (auth, products)
  shared/
    api/          # Axios instance, API функции
    ui/           # Переиспользуемые UI компоненты (Button, Input, Table, etc.)
    lib/          # Утилиты, хелперы
    types/        # Общие типы
```
